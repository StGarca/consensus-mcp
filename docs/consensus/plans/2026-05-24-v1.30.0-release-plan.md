# v1.30.0 Release Plan — cross-AI weighting, cost-tiers, operator-declared rigor

**Goal:** ship the weighting + cost-tiers + operator-declared-rigor + user-centric-lean
system as **v1.30.0** (minor), **activated behind off-by-default flags** — defaults are
byte-identical to current behavior; the operator opts in. (Per the SWOT/next-step
consults: do NOT release inert dead code, and never auto-decide a bias — so v1.30.0 ships
*usable, opt-in* machinery, not a silent behavior change.)

**Base:** `main` is at v1.29.5 (the integration guard already shipped). Work lives on
`iter-advisory-weights-2026-05-24` + `iter-operator-declared-rigor-2026-05-25` (off
v1.29.4), all UNMERGED.

## Scope — what's IN v1.30.0

Already built + tested (pure cores, on the two branches):
- **Operator-declared rigor tiers** — `effective_tier` (authoritative, raises on no
  declaration — NO inference), `suggest_tier` (advisory only), `governance_floor` (sole
  monotone auto-move: DEEP+locked), `estimate_cost` (pre-commit band).
- **Advisory contributor weighting** — discount-only, hard floor, same-family cap,
  no-self-grade firewall (write AND read); `order_proposal_paths` (stable permutation →
  weights-off equivalence, proven at the engine level).
- **External outcome ledger** (append-only, AI-adjudicator-rejecting) + **Beta learner
  repurposed as a SCORECARD** (`build_scorecard`, decision-support only) + **A/B
  beats-uniform** revert-gate.
- **User-centric AI lean** — `lean_to_weights` (operator-DECLARED ranking → synthesis
  reading-order); the machine measures, the operator declares the bias.
- **External-CLI retry/fallback policy** (pure decision core).
- **Per-iteration telemetry** (cost-per-blocking-finding by tier).
- **interaction_surface validator**; **routing decision-table doc**; **skill codification**.
- **Governed-project integration guard** — already shipped in v1.29.5.

## Remaining work before the cut (the "activation" iteration — all flagged OFF by default)

- **A. Live-engine wiring (behind off-by-default flags):**
  1. Read the operator-declared tier from the goal_packet (via the `_goal_risk_class`
     path; unify under `risk_class`) → route through `effective_tier`. Flag: off ⇒
     current behavior.
  2. Thread the operator-declared lean into `_build_convergence_packet`'s call site in
     the engine round loop (the packet already accepts `contributor_weights`). Flag: off ⇒
     identity order.
  3. Wire the retry/fallback policy into `_dispatch_base` (env flag
     `CONSENSUS_MCP_RETRY_POLICY`, default off; the kimi integrity-false-positive path
     bracketed by the existing snapshots).
- **B. Verification harnesses (external; gate every flag before it can default on):**
  weights-off/flags-off **replay over an archived corpus** (byte-identical gate verdicts);
  governed-project **tier-router smoke**; **A/B-beats-uniform** (learner stays
  scorecard-only — no auto-weight ships in v1.30.0); a **flags-off default contract test**.
- **C. Scorecard surfacing** — expose `build_scorecard` via `consensus results` (or a new
  read-only command) so the operator sees performance history to declare the lean from.
- **D. Docs/CHANGELOG/README** for v1.30.0 (the opt-in flags, operator-declared tiers,
  the scorecard→lean UX, the no-self-grade + weights-off + no-inference invariants).
