# v1.19.0 - Result logging (converged design)

**Status:** converged via 4-way Workflow A consult (claude + codex + gemini +
kimi), iteration `result-logging-scope-2026-05-22`. This is the implementation
spec. (Workflow B review skipped per operator for this change - A suffices.)

## Problem
consensus-mcp seals every review pass for provenance, but run RESULTS aren't
recorded in a consistent, queryable form: findings live as structured pass items,
synthesized converged-plan items, and prose; fixes/dispositions aren't events
(`fixes_applied`=0 everywhere); formats differ across projects/eras. So findings
can't be aggregated into a trustworthy project scorecard.

## Decision (weighted-synthesis)

### Storage
- **Authoritative ledger:** `consensus-state/state/results-v1.jsonl` - durable,
  project-local, **gitignored** (`.gitignore` already excludes
  `consensus-state/state/*.jsonl`), one current snapshot per `iteration_id`,
  `consensus_results_schema_version` on every record. Survives cleanup of the
  gitignored `active/iteration-*/` dirs (which the per-iteration-only approach
  would lose). One-file rollup.
- **Co-located detail:** `consensus-state/active/<iteration_id>/iteration-results.yaml`
  - human-readable per-iteration record (gitignored via `active/iteration-*/`).

### Schema (`schemas/results-v1.schema.json`, schema_version: 1)
Per-iteration snapshot: `iteration_id`, `run_id`, dates, `reviewers[{name,family}]`,
`convergence{rule,converged,goal_satisfied_by}`,
`findings[{id, severity, source_reviewer, source_pass_id, disposition, evidence_ref?, fix?}]`
where `disposition  in  {validated_fixed, dismissed_refuted, deferred, open}`,
`evidence_ref` (commit/experiment) required for `dismissed_refuted`, `fix{patch_id,files}`
for `validated_fixed`; `counts{by_severity, validated, dismissed, deferred, fixes_applied}`;
`source_audit_event_id` + `source_audit_yaml_post_sha256` (provenance link); dedup
findings by `id` within an iteration.

### Write path
- A new `_results_log.py` authors the per-iteration record + updates the JSONL
  ledger (atomic rewrite of the one snapshot for that `iteration_id`,
  schema-validate before replace).
- `tools/audit_append_event.py` `iteration_closed` handler calls it AFTER the
  closure invariant + closure-certificate, following the existing
  non-fatal-but-warns pattern (surface errors; never silently skip; reuse the
  single-writer lock).
- **Disposition + fix captured as structured audit fields at event time** (the
  fix for `fixes_applied`=0): `apply_step_landed += {finding_ids, files_touched,
  fix_summary}`; `iteration_closed += {finding_dispositions:[{id, disposition,
  evidence_ref}]}`. The close-time author cross-references sealed-pass findings x
  apply events x closure dispositions. New fields are ADDITIVE/optional -
  backward-compatible with existing events + the 977-test suite.

### Rollup
- `consensus results` (console-script + MCP tool; logic in `_results_rollup.py`):
  reads `results-v1.jsonl` -> project scorecard (findings by severity, validated
  vs dismissed-with-evidence vs deferred, fixes applied, iterations, convergence
  rate, date span). Human table + `--json`. Read-only. Skips/warns records with
  unknown `schema_version`.

### Backfill
Forward-only is authoritative. Optional `--backfill --best-effort` populates
findings-RAISED from existing sealed passes, every such record flagged
`backfilled: true, confidence: best-effort` and REPORTED SEPARATELY by the rollup
so it never contaminates the trustworthy forward count. Default = forward-only.

## Files

- tests for each
- `.gitignore` already covers the storage (verify, no change expected)

## Acceptance gates
- Closing an iteration writes a schema-valid record to `results-v1.jsonl` (one
  snapshot/iteration) + a co-located `iteration-results.yaml`; both gitignored.
- `apply_step_landed` + `iteration_closed` carry the new fields; `fixes_applied`
  is > 0 when fixes happen.
- `consensus results` matches hand counts on a fixture project; `--json` works.
- Findings deduped by id within an iteration.
- Backfill records flagged + reported separately; default forward-only.
- Unknown `schema_version` records skipped with a warning.
- **Existing 977-test suite stays green** (audit-event changes are additive).
